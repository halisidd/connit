🔐 Secure VM & Reverse Proxy Deployment Checklist

🧩 Common Security Steps (All VMs)

  • SSH Hardening – Disable root login, use key auth, restrict IPs
  • UFW (Firewall) – Allow only necessary ports
  • Fail2Ban – Block brute force attacks (esp. SSH)
  • Automatic Security Updates – unattended-upgrades + autoremove
  • Disable Unused Services – Minimize attack surface
  • System Monitoring (Netdata, etc.) – Resource + network visibility
  • Regular Cleanup – Remove unused packages, old kernels, logs
  • Template Maintenance – Boot, update, shut down monthly or via cron

🖥️ New VM Deployment (Specific)

  • Create limited user (non-root) – Use sudo for admin tasks
  • SSH Port Restriction – Change default port (optional)
  • Set hostname & FQDN – Easier identification
  • Setup log rotation – Keep disk usage clean
  • Test update script manually first – Before automating
  • Keep templates updated – Use Proxmox script or cron

🌐 Reverse Proxy Server Setup (Specific)

  • Only expose 80/443 + SSH – Block all other ports
  • HTTPS Only – Use Let’s Encrypt & redirect HTTP to HTTPS
  • Enable Security Headers – HSTS, CSP, X-Frame-Options, etc.
  • Add Auth for Admin Interfaces – Protect Nginx Proxy Manager or UIs
  • IP Whitelisting for admin access – Optional but adds security
  • Monitor proxy logs – Watch failed attempts or abuse
  • Don't host other apps on proxy – Keep it lean & hardened
  • Keep NPM/Nginx updated – Apply patches, renew certs
  • Back up config & SSL certs – For disaster recovery
  • Use proxy even for internal-only apps – Add HTTP auth if needed
